## How to Talk to an Agent

### Where to talk

Each agent lives in its **dedicated Discord channel**. Just type your message in the channel and the agent will respond. You do **not** need to @mention the agent in its own channel -- it's always listening there.

If you want to reach an agent from a different channel, use `@AgentName` to tag them.

### Use threads for parallel conversations

Multiple people can talk to the same agent at the same time. If someone else is already chatting with Mila in the #mila channel and you also need her, **start a Discord thread**. The agent will follow each thread as a separate conversation, so your requests won't mix with someone else's. This also keeps the main channel clean -- especially useful when you're going back and forth with iterations.

### How to phrase requests

The agents understand natural language. Talk to them like you would a colleague. That said, being specific gets you better results.

**Good:**
- "Create a Twitter thread about the Lynex epoch 47 revenue numbers. Highlight the 12% growth."
- "What's our PTO policy? How many days do I have if I started in March?"
- "Design a banner for the Vaultedge launch on Base. Use the Vaultedge brand colors."

**Less good:**
- "Do the thing" (what thing?)
- "Make it better" (better how?)
- "Content" (about what, for where?)

### "Propose" vs "Do" -- this matters

Agents treat these differently:

- **"Propose X"** / **"Suggest X"** / **"Outline X"** = the agent presents a plan and **waits for your approval** before doing anything.
- **"Do X"** / **"Create X"** / **"Execute X"** = the agent **goes ahead and does it**.

This is powerful. You can say "Propose a campaign strategy" to see the plan first, review it, tweak it, and then say "Go ahead" to execute. Or say "Create the campaign" if you trust the agent to just run with it.

### They'll ask you questions too

Agents don't just take orders -- they ask clarifying questions when they need more context. They might ask about your preferences, who the audience is, what tone you want, or what success looks like. Answer naturally -- these answers help them do better work and they'll remember your preferences for next time.

They also proactively surface ideas you haven't thought of. If an agent says "I noticed X, want me to do something about it?" -- that's a feature, not a bug.

### Conversation tips

- **Be direct.** Skip pleasantries. "Create a tweet about X" beats "Hey, could you maybe help me with something?"
- **Give context.** The more detail you provide, the better the output. Include numbers, links, target audience, tone preferences.
- **Iterate.** If the first output isn't right, tell the agent what to change: "Make it shorter", "More professional tone", "Add the TVL numbers".
- **Ask follow-ups.** Agents remember the conversation. You can say "Now make a LinkedIn version of that" without repeating everything.
- **Multi-step is fine.** You can ask agents to plan something first, then execute: "Draft a campaign plan for Vaultedge launch, then create the social posts."
- **Correct them.** When an agent gets something wrong, tell it directly: "That's wrong, the correct number is X." Corrections are one of the strongest learning signals -- the agent retains the correction and applies it going forward.
- **You can DM agents.** Besides talking in their channel, you can also send agents direct messages on Discord for private conversations.

## What Makes These Agents Special

### They remember

Every agent has persistent memory. If you tell GregAIlia that Tomas prefers async check-ins, she'll remember that next time. If you correct Mila's tone on a tweet, she'll adjust going forward. They learn from every interaction.

You can explicitly tell agents to remember things:
- "Remember that we always capitalize USDVE"
- "Keep in mind that the launch date moved to March 20"

And you can ask them to recall past context:
- "What did we discuss about the Vaultedge marketing plan?"
- "Recall the feedback from our last session"

### They can use tools

These aren't just chatbots. Depending on the agent, they can:
- **Browse the web** (research competitors, check live data, scrape DeFi dashboards)
- **Create files** (presentations, reports, banners, graphics)
- **Access Google Drive** (read and create docs, upload files)
- **Manage tasks in ClickUp** (create, update, track tasks)
- **Generate images** (AI-generated visuals, backgrounds, brand assets)
- **Render graphics** (pixel-perfect banners from HTML/CSS templates)

### They collaborate with each other

Agents can talk to each other. If you ask Mila to create a marketing campaign and she needs a banner, she can ask Miro to design it. If Carl needs budget approval for a partnership deal, he can check with Aria. You can also explicitly ask an agent to loop in another one.

### They work proactively

Agents don't just respond -- they also think ahead. They might notice patterns in your requests, suggest improvements, or flag things you haven't thought of. This behavior will improve over time as they learn more about the team and workflows.

Importantly: agents always **propose before acting** on proactive ideas. They'll say "I noticed X, want me to do Y?" rather than just doing it. Nothing goes external (tweets, emails, messages) without your approval.

### They ask permission for risky actions

Agents will ask for confirmation before doing things that affect the outside world: sending messages to external platforms, making API calls, or executing destructive operations. If an agent asks "Should I go ahead with this?" -- it's because the action has real consequences.

### Long conversations

In very long conversations, agents may start to lose older context from the beginning of the chat. This is normal. The important stuff gets saved to their memory automatically, so they won't lose critical facts. If an agent seems to have forgotten something from earlier, just remind it or ask it to recall: "Check your memory about the Vaultedge launch plan."

## Agents Working Together

One of the most powerful features is chaining agents. Here are some real workflows:

**Launch campaign (multi-agent):**
1. Ask **Mila** to create the campaign strategy and social copy
2. Ask **Miro** to design the visuals Mila needs
3. Ask **HATZO** to plan the community engagement around the launch
4. Ask **Carl** to prepare partner outreach materials
5. Ask **Aria** to track the campaign budget

**Team event (multi-agent):**
1. Ask **GregAIlia** for the team roster and upcoming birthdays
2. Ask **HATZO** to plan a community celebration
3. Ask **Miro** to create a birthday graphic

**New partnership (multi-agent):**
1. Ask **Carl** to research the partner and create a proposal
2. Ask **Mila** to draft the announcement copy
3. Ask **Miro** to design the co-branded graphics
4. Ask **Aria** to model the financial impact

You can also explicitly tell an agent to involve another: "Mila, create a tweet thread for the Lynex partnership and ask Miro to make a banner for it."

## What to Expect in This Test Phase

- **Response quality will vary.** Some requests will nail it on the first try, others will need iteration. That's normal.
- **Memory improves over time.** The more you interact, the better agents understand your preferences and context.
- **Things might break.** If an agent stops responding or gives weird output, let the team know.
- **Your feedback is gold.** Tell agents when they get it wrong. Correct their tone, their facts, their approach. They learn from corrections.
- **Be creative.** Push the boundaries of what you think they can do. The advanced capabilities (web browsing, file creation, cross-agent collaboration) are real -- test them.
